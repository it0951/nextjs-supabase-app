# 모이자 (Moiza) MVP PRD

## 🎯 핵심 정보

**목적**: 소모임 주최자의 반복 작업(공지, 참여자 관리, 카풀 배정, 정산)을 하나의 공간에서 처리
**사용자**: 친구 모임/동호회/동아리 등 소규모 이벤트를 주최하거나 참여하는 모든 사람

---

## 🚶 사용자 여정

```
[주최자 여정]

1. 로그인 페이지
   ↓ Google 로그인 버튼 클릭

2. 대시보드 페이지
   ↓ "이벤트 만들기" 버튼 클릭

3. 이벤트 생성 페이지
   ↓ 제목/날짜/장소/최대 인원 입력 후 "생성" 클릭

4. 이벤트 관리 페이지 (주최자 전용)
   ├── "초대 링크 복사" → 카톡/문자로 링크 공유
   ├── "공지 작성" 탭 → 공지 작성/수정/삭제
   ├── "참여자" 탭 → 참여 신청 확인, 확정/취소/대기 상태 변경
   ├── "카풀" 탭 → 운전자 그룹 생성, 참여자 배정
   └── "정산" 탭 → 항목 입력, 개인별 금액 확인, 납부 여부 처리

---

[참여자 여정]

1. 이벤트 초대 페이지 (초대 링크 클릭)
   ↓ 이벤트 정보 확인 후 "참여 신청" 클릭

   [신규 참여자]         → 이름+연락처 입력 → 참여 신청 완료 → 참여자 뷰 페이지
   [이미 등록된 참여자] → 이름+연락처 입력 → 본인 확인     → 참여자 뷰 페이지

2. 참여자 뷰 페이지 (join_token으로 접근)
   ├── 공지 탭 → 공지사항 목록 확인
   ├── 카풀 탭 → 본인 배정된 카풀 그룹 확인
   └── 정산 탭 → 본인 납부 금액 확인
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명              | 설명                                                        | MVP 필수 이유                         | 관련 페이지                            |
| -------- | ------------------- | ----------------------------------------------------------- | ------------------------------------- | -------------------------------------- |
| **F001** | 이벤트 생성/수정    | 제목, 날짜, 장소, 최대 인원 입력으로 이벤트 생성            | 서비스의 출발점, 모든 기능의 컨테이너 | 이벤트 생성 페이지, 이벤트 관리 페이지 |
| **F002** | 초대 링크 생성/공유 | invite_token 기반 고유 링크 생성 및 클립보드 복사           | 참여자 모집의 유일한 수단             | 이벤트 관리 페이지                     |
| **F003** | 공지 작성/관리      | 공지 작성, 수정, 삭제 및 목록 표시                          | 이벤트 정보 전달의 핵심 채널          | 이벤트 관리 페이지, 참여자 뷰 페이지   |
| **F004** | 참여자 등록         | 비회원이 이름+연락처 입력으로 참여 신청                     | 비회원 참여 구조의 핵심 진입점        | 이벤트 초대 페이지                     |
| **F005** | 참여 상태 관리      | 확정/대기/취소 상태 변경, 참여자 목록 조회                  | 인원 관리의 핵심 기능                 | 이벤트 관리 페이지                     |
| **F006** | 카풀 그룹 배정      | 운전자+좌석수로 그룹 생성, 참여자 수동 배정                 | 카풀 조율 자동화, 반복 소통 제거      | 이벤트 관리 페이지, 참여자 뷰 페이지   |
| **F007** | 항목별 정산         | 항목(식비/장소비 등) 입력, 참여자 균등 분배, 납부 여부 관리 | 비용 정산 투명화, 반복 계산 제거      | 이벤트 관리 페이지, 참여자 뷰 페이지   |

### 2. MVP 필수 지원 기능

| ID       | 기능명           | 설명                                                          | MVP 필수 이유                             | 관련 페이지                          |
| -------- | ---------------- | ------------------------------------------------------------- | ----------------------------------------- | ------------------------------------ |
| **F010** | 주최자 인증      | Google 소셜 로그인/로그아웃, 이메일+비밀번호 직접 가입/로그인 | 이벤트 관리 권한 구분에 필요한 최소 인증  | 로그인 페이지, 회원가입 페이지       |
| **F011** | 이벤트 목록 관리 | 주최자 기준 이벤트 목록 조회, 이벤트 삭제                     | 주최자가 여러 이벤트를 관리하기 위한 허브 | 대시보드 페이지                      |
| **F012** | 참여자 본인 확인 | join_token으로 참여자 뷰에 재진입 (이름+연락처 입력)          | 비회원 구조에서 본인 데이터 접근 수단     | 이벤트 초대 페이지, 참여자 뷰 페이지 |

## 📄 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F010` | **인증:** 불필요 (공개 페이지)

| 항목            | 내용                                                                                                |
| --------------- | --------------------------------------------------------------------------------------------------- |
| **역할**        | 주최자 전용 인증 진입점                                                                             |
| **진입 경로**   | 최초 접속, 로그아웃 후 자동 리디렉션, 비인증 상태에서 보호 페이지 접근 시                           |
| **사용자 행동** | Google 로그인 버튼 클릭으로 OAuth 인증 진행                                                         |
| **주요 기능**   | Google OAuth 로그인 버튼, 로그인 성공 시 대시보드로 이동, 이미 로그인된 경우 대시보드 자동 리디렉션 |
| **다음 이동**   | 성공 → 대시보드 페이지, 실패 → 에러 메시지 표시 후 로그인 페이지 유지                               |

---

### 회원가입 페이지

> **구현 기능:** `F010` | **인증:** 불필요 (공개 페이지)

| 항목            | 내용                                                                                                                                    |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 이메일+비밀번호 방식으로 신규 주최자 계정 생성                                                                                          |
| **진입 경로**   | 로그인 페이지의 "회원가입" 링크 클릭                                                                                                    |
| **사용자 행동** | 이메일, 비밀번호, 비밀번호 확인 입력 후 가입 버튼 클릭                                                                                  |
| **주요 기능**   | 이메일 입력 필드, 비밀번호 입력 필드, 비밀번호 확인 입력 필드, 비밀번호 불일치 시 에러 메시지 표시, **이메일 인증 없이 즉시 가입 완료** |
| **다음 이동**   | 성공 → 대시보드 페이지 (즉시 이동, 인증 이메일 없음), 실패 → 에러 메시지 표시 후 회원가입 페이지 유지                                   |

> **참고**: Supabase 대시보드에서 `Authentication → Providers → Email → Confirm email`이 비활성화되어 있어야 합니다. 비활성화 상태에서는 가입 즉시 세션이 생성되어 바로 대시보드로 이동합니다.

### 대시보드 페이지

| 항목            | 내용                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 주최자의 이벤트 관리 허브, 전체 이벤트 현황 파악                                                                                      |
| **진입 경로**   | Google 로그인 성공 후 자동 이동, 헤더 로고 클릭                                                                                       |
| **사용자 행동** | 본인이 만든 이벤트 목록 조회, 이벤트 클릭으로 관리 페이지 이동, 새 이벤트 생성 버튼 클릭                                              |
| **주요 기능**   | 이벤트 카드 목록 (제목, 날짜, 장소, 확정 참여자 수 표시), **"이벤트 만들기"** 버튼, 이벤트 삭제 버튼 (확인 다이얼로그), 로그아웃 버튼 |
| **다음 이동**   | 이벤트 클릭 → 이벤트 관리 페이지, 이벤트 만들기 → 이벤트 생성 페이지                                                                  |

| 항목            | 내용                                                                                                                                                                   |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 새 이벤트 생성 전용 폼 페이지                                                                                                                                          |

| **주요 기능**   | **[개요 탭]** 이벤트 정보 표시 및 수정 (F001), **"초대 링크 복사"** 버튼 (F002) / **[공지 탭]** 공지 작성 폼, 공지 목록, 수정/삭제 버튼 (F003) / **[참여자 탭]** 참여자 목록(이름, 연락처, 상태), 상태 변경 드롭다운 (확정/대기/취소), 참여자 삭제 버튼 (F005) / **[카풀 탭]** 운전자 그룹 생성 (운전자 지정+좌석수 입력), 미배정 참여자 목록, 선택으로 그룹에 배정 (F006) / **[정산 탭]** 항목 추가 (항목명+금액), 확정 참여자 기준 자동 균등 분배 금액 표시, 납부 여부 체크박스 (F007) |

### 이벤트 초대 페이지

> **구현 기능:** `F004`, `F012` | **인증:** 불필요 (invite_token으로 접근)

| 항목            | 내용                                                                                                                                                                                                                                                    |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 비회원 참여자의 이벤트 진입 및 참여 신청 페이지                                                                                                                                                                                                         |
| **진입 경로**   | 주최자가 공유한 초대 링크 클릭 (invite_token 포함 URL)                                                                                                                                                                                                  |
| **사용자 행동** | 이벤트 정보 확인 후 이름+연락처 입력으로 참여 신청 또는 기존 참여자로 본인 확인                                                                                                                                                                         |
| **주요 기능**   | 이벤트 기본 정보 표시 (제목, 날짜, 장소, 현재 확정 인원/최대 인원), 이름 입력 필드 (필수), 연락처 입력 필드 (필수), **"참여 신청"** 버튼 (신규: 참여자 등록, 기존: 본인 확인), 최대 인원 초과 시 대기 신청 안내, 유효하지 않은 토큰 시 에러 페이지 표시 |
| **다음 이동**   | 성공 (신규/기존 확인) → 참여자 뷰 페이지 (join_token 발급), 유효하지 않은 링크 → 에러 안내                                                                                                                                                              |

### 참여자 뷰 페이지

> **구현 기능:** `F003`, `F006`, `F007`, `F012` | **인증:** join_token으로 접근 (로그인 불필요)

| 항목            | 내용                                                                                                                                                                                                                                                                                       |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **역할**        | 참여자가 이벤트 관련 정보를 조회하는 읽기 중심 뷰                                                                                                                                                                                                                                          |
| **진입 경로**   | 이벤트 초대 페이지에서 참여 신청/본인 확인 성공 후 join_token URL로 이동                                                                                                                                                                                                                   |
| **사용자 행동** | 공지 확인, 본인 카풀 배정 확인, 본인 납부 금액 확인                                                                                                                                                                                                                                        |
| **주요 기능**   | **[공지 탭]** 공지사항 목록 조회 (최신순, 읽기 전용) (F003) / **[카풀 탭]** 본인이 배정된 카풀 그룹 표시 (운전자명, 동승자 목록), 미배정 안내 (F006) / **[정산 탭]** 본인 부담 항목별 금액 및 합계 표시, 납부 여부 상태 표시 (F007) / 이벤트 기본 정보 헤더 표시 (제목, 날짜, 장소) (F012) |
| **다음 이동**   | 탭 전환 (페이지 내 이동), join_token 만료/무효 → 이벤트 초대 페이지로 안내                                                                                                                                                                                                                 |

## 📱 UI/UX 명세 (모바일 퍼스트)

### 설계 원칙

- **모바일 퍼스트**: 375px(iPhone SE) 기준으로 먼저 설계, 상위 해상도는 확장
- **앱 같은 경험**: 상단 고정 헤더 + 하단 고정 탭 네비게이션으로 네이티브 앱 UX 구현
- **터치 우선**: 모든 인터랙션 요소 최소 44px 터치 타겟 확보
- **단일 컬럼**: 모바일 기본은 1열 세로 스크롤, 콘텐츠 계층 명확화

### 공통 레이아웃 구조

```
┌─────────────────────────────┐  ← fixed top-0, h-14
│  🗓 모이자        🔔  👤  │  (헤더: 로고 + 알림 + 프로필)
├─────────────────────────────┤
│                             │  ← pt-14 pb-20, px-4
│    (페이지 콘텐츠 영역)      │  (세로 스크롤)
│                             │
├─────────────────────────────┤  ← fixed bottom-0, h-16

└─────────────────────────────┘
```
