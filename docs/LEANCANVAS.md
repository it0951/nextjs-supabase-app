## 세부 내용

### 1. 문제 (Problem)

| #   | 문제                                                     | 현재 대안                       |
| --- | -------------------------------------------------------- | ------------------------------- |
| P1  | 공지·참여자·카풀·정산을 여러 채널에서 분산 관리          | 카카오톡 단체 채팅방, 엑셀 시트 |
| P2  | 매번 수동으로 반복되는 인원 확인·카풀 조율·더치페이 계산 | 직접 계산, N/1 앱 별도 사용     |
| P3  | 참여자가 앱 설치·회원가입 없이 모임 정보 확인 불가       | 링크 없이 카톡으로 개별 안내    |

### 2. 고객 세그먼트 (Customer Segments)

**얼리어답터 (Early Adopters)**

- 월 1회 이상 정기 모임을 운영하는 소모임 주최자
- 5~20명 규모의 소규모 그룹을 관리하는 동아리·동호회 운영자

### 3. 가치 제안 (Unique Value Proposition)

**주최자**: "이벤트 만들고 링크 공유하면 끝 — 공지·참여자·카풀·정산까지"  
**참여자**: "링크 클릭 한 번으로 참여 등록, 카풀·정산 정보 바로 확인"

### 4. 솔루션 (Solution)

| 기능        | 설명                                                  |
| ----------- | ----------------------------------------------------- |
| 이벤트 생성 | 제목·날짜·장소·인원 입력 후 초대 링크 자동 생성       |
| 비회원 참여 | 이름+연락처만으로 링크 클릭 즉시 참여 등록            |
| 참여자 관리 | 확정/대기/취소 상태 관리, 인원 초과 시 대기 자동 처리 |
| 카풀 배정   | 주최자가 운전자·동승자 그룹 수동 배정                 |
| 항목별 정산 | 식비·장소비 등 항목 입력 → 인원수 균등 분배 자동 계산 |

### 5. 채널 (Channels)

- **1순위**: 초대 링크를 카카오톡·문자로 공유 (서비스 자체가 채널)
- **2순위**: 소모임 커뮤니티(네이버 카페, 오픈채팅) 입소문
- **3순위**: SEO (모임 관리 앱, 더치페이 계산기 키워드)

### 6. 수익 흐름 (Revenue Streams)

| 단계 | 수익 모델          | 비고                                  |
| ---- | ------------------ | ------------------------------------- |
| MVP  | 무료               | 사용자 확보 및 검증                   |
| v2   | 월 구독 (프리미엄) | 무제한 이벤트, 송금 링크 등 고급 기능 |
| v3   | B2B 단체 요금제    | 동호회·기업 동아리 대상 팀 플랜       |

### 7. 비용 구조 (Cost Structure)

| 항목   | 내용                                |
| ------ | ----------------------------------- |
| 인프라 | Vercel (호스팅), Supabase (DB·Auth) |
| 개발   | 초기 개발 및 유지보수               |
| 도메인 | 연간 도메인 비용                    |

### 8. 핵심 지표 (Key Metrics)

| 지표                    | 설명                            |
| ----------------------- | ------------------------------- |
| 생성된 이벤트 수        | 서비스 활용도의 핵심 지표       |
| 월 활성 주최자 수 (MAU) | 서비스 지속 사용 여부           |
| 초대 링크 전환율        | 링크 클릭 → 참여 등록 완료 비율 |
| 이벤트당 평균 참여자 수 | 모임 규모 파악                  |
| 정산·카풀 기능 사용률   | 핵심 기능 채택률                |

### 9. 경쟁우위 (Unfair Advantage)

| 우위             | 설명                                             |
| ---------------- | ------------------------------------------------ |
| 비회원 참여 구조 | 회원가입 없이 링크만으로 참여 — 진입 장벽 최소화 |
| 올인원 통합      | 공지·참여자·카풀·정산을 하나의 서비스에서 처리   |
| 링크 기반 공유   | 별도 홍보 없이 초대 링크 자체가 바이럴 채널      |
